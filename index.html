「Webコンテンツ作成の成果」

本テーマの学習目標

  • HTML によって Web ページの構造を記述する基本的な方法がわかる
  • CSS によって Web ページの見た目を設定する基本的な方法がわかる
  • JavaScript によって Web ページのインタラクティブな処理を記述する基本的な方法がわかる
  • HTML, CSS, JavaScript の構文リファレンスから，構文の書式や使用方法の情報を入手できる
  • HTML, CSS, JavaScript, 画像・音声・動画ファイルを組み合わせて簡単な Web ページが作成できる

プロジェクト課題について

プロジェクト課題では、「世界の天気」を実施する。
リストにデータを表示できる都市名を一覧にし、その中から都市名をひとつ選んで検索ボックスに入力すると、その都市の天気・最高気温・最低気温・湿度・風速が表示されます。また、都市名は人の手で入力されるので、文字の打ち間違えなどがあった場合の表示についても考えました。

工夫点，苦心点，強調点，感想

私はもともとWebデザインに興味があり、この授業でWeb制作を学ぶ前から少しですがＨＴＭＬやＣＳＳについて自分で勉強していたので授業の初めは知っていた内容も多く、そこまで難しいと感じることはありませんでした。しかし、授業が進みJavaScriptなどの動的な処理を組み込むようになってくると一気に難易度が増しなかなか授業のペースについていけないことも多々ありました。
なので、苦心点は数えきれないほどありました。プロジェクト課題ではJavaScriptとHTMLがうまくかみ合わず、データが表示されなかったり思った通りの動きをしなかったりと、仕組みを理解するまでにかなりの時間がかかってしまいました。ページ上での工夫ではありませんが、JavaScriptとHTMLの間でidや変数名が分からなくならないようにHTMLには好きな芸能人の苗字を、JavaScriptには名前を使いどこが繋がっているのかを分かりやすくしたことが工夫点です。また、JavaScriptとJavaで書き方が共通している所とそうでないところがあり、プログラムの作成中に悩むことが多かったです。
レポート課題ではまずこんなに長くコードを書いたのが初めてだったので途中から自分でもどこまで作業しているのか分からなくなることがありました。また、これだけ文字のあるページをどうしたら飽きずに見やすいページにできるかというところで悩みました。それでもできるだけ各回ごとに説明のまとめ方をそろえてみたり、リンクや見出しに色を使ってみたりと自分なりに工夫してみました。私はデザインにも少し興味があったので、正直CSSで文字のどの色を考えているのが一番楽しかったです。
とにかく毎週分からないことだらけで、やっぱりWeb制作は奥が深くて面白いと感じていた半面、自分は本当にこの習った内容ができるようになるのかという不安もありました。それでも先生や先輩、周りの友人の助けもあり、授業のたびに楽しさを感じながらなんとか最後まで取り組むことができました。友人と一緒にやることでWebにはそれぞれ個性が出るということを改めて実感しました。また、周りの人に助けられながらでも以前よりプログラムの仕組みが見て分かるようになったことがうれしかったです。今回の授業で学んだことは本当に将来役に立つことだと思うし、自分が興味を持っていた分野なのでこれからさらにつきつめていければと思います。
